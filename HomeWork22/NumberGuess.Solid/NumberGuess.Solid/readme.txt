Принцип единственной ответственности
Каждый класс выполняет одно действие
  * GameEngine - отвечает за сценарии игры
  * GuessValuer - отвечает за сравние чисел
  * RandomNumberGenerator - отвечает за генерацию случайного числа
  * ConsoleInputReader - отвечает за чтение данных из консоли
  * ConsoleOutputWriter - отвечает за вывод сообщений на консоль
  * JsonGameSettings - отвечает за чтение/валидацию настроек
   
Принцип инверсии зависимостей
Реализация зависит от абстаркции(интерфесы). нет зависимости от классов/ к примеру GameEngine
 
Принцип разделения интерфейса
  * Интерфейсы сделаны маленькими и узкими. Какжды интерфейс отвечает за что то одно. Нужно для уменьшения связанности 
  и легче производить замену.
  
Принцип открытости/закрытости
 * Можно поменять логику без изменения  GameEngine. К примеру хотим изменить вывод на вывод в файл то реализуем IOutputWriter  
 
Принцип подстановки Барбары Лисков
 * Реализацию любого интерфейса можно заменить без всяких проблем и игра не сломается. Обеспечивается через контракты интерфейсов
